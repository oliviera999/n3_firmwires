# Fix CI/CD GitHub Actions - 2025-10-12

## 🔴 Problème Initial

Le build GitHub Actions échouait systématiquement avec les erreurs suivantes :

```
build
Process completed with exit code 1.
build
The process '/usr/bin/git' failed with exit code 128
```

## 🔍 Diagnostic

### Erreur 1 : Exit Code 1 (Build Failure)

**Cause identifiée** : Le workflow utilisait un environnement PlatformIO inexistant `esp32dev`

```bash
# Test local
pio run --environment esp32dev
# Erreur: Unknown environment names 'esp32dev'. Valid names are 'wroom-dev, wroom-prod, ...'
```

**Environnements valides** dans `platformio.ini` :
- `wroom-dev` : Développement avec debug
- `wroom-prod` : Production optimisée  
- `wroom-prod-pio6` : Production avec PIO6
- `wroom-test`, `s3-prod`, `s3-dev`, etc.

### Erreur 2 : Exit Code 128 (Git Failure)

**Cause identifiée** : Fichier `.gitmodules` mal configuré

```ini
# Ancien .gitmodules (INCORRECT)
[submodule "ffp3distant"]
	path = ffp3distant
	url = https://github.com/oliviera999/ffp3distant.git
```

**Problèmes** :
- `ffp3distant` n'existe plus dans le projet (déplacé ou supprimé)
- Le sous-module `ffp3` existe mais n'était pas déclaré dans `.gitmodules`
- Git tentait de cloner un sous-module inexistant → Exit code 128

## ✅ Solutions Appliquées

### 1. Correction de `.gitmodules`

```ini
# Nouveau .gitmodules (CORRECT)
[submodule "ffp3"]
	path = ffp3
	url = https://github.com/oliviera999/ffp3.git
```

**Actions** :
- ✅ Suppression de la référence à `ffp3distant`
- ✅ Ajout de la référence correcte à `ffp3`
- ✅ Synchronisation : `git submodule sync`

### 2. Création du workflow GitHub Actions

**Fichier** : `.github/workflows/build.yml`

**Caractéristiques principales** :

#### a) Environnements corrects
```yaml
strategy:
  matrix:
    environment: [wroom-prod, wroom-dev]
```

#### b) Gestion des sous-modules
```yaml
- name: Checkout code
  uses: actions/checkout@v4
  with:
    submodules: false  # Désactivé pour éviter les erreurs
    fetch-depth: 0
```

#### c) Cache optimisé
```yaml
- name: Cache PlatformIO
  uses: actions/cache@v4
  with:
    path: ~/.platformio
    key: ${{ runner.os }}-platformio-${{ hashFiles('**/platformio.ini') }}
```

**Gains** :
- 🚀 Réduction du temps de build de ~2 minutes
- 💰 Économie de ressources GitHub Actions
- 📦 Cache automatique des dépendances

#### d) Artifacts
```yaml
- name: Archive firmware
  uses: actions/upload-artifact@v4
  with:
    name: firmware-${{ matrix.environment }}
    path: |
      .pio/build/${{ matrix.environment }}/firmware.bin
      .pio/build/${{ matrix.environment }}/firmware.elf
    retention-days: 30
```

**Bénéfices** :
- 📥 Téléchargement direct du firmware depuis GitHub
- 🔍 Conservation pour debugging (30 jours)
- 🎯 Firmwares pour chaque environnement séparément

### 1. Sous-modules dans `unused/`

```

```

**Recommandation** : Ces dossiers `unused/` contiennent des sous-modules imbriqués qui posent problème.

**Options** :

- **Option B** : Les supprimer complètement

**Action recommandée** : Option A (gitignore)

